[2.4] En mi caso, al parar el video en el VLC emisor tarda entre 2 y 3 segundos en parar el receptor la imagen, el mismo tiempo se observa cuando se reanuda la emisión. Este retraso es debido a tres causas:

·       Decodificación/Codificación de VLC. Tiempo necesario para codificar y decodificar los datos en ambos reproductores (emisor y receptor).

·       Latencia de la red. El tiempo que tardan los datos en viajar del emisor al receptor, en este caso son mínimos por no haber una infraestructura de red en medio, aunque mínimo no descartable por consumir recursos del propio equipo de trabajo.

·       Buffering de datos. El receptor dispone de una caché de buffer con los datos que recibe y sigue reproduciendo el contenido almacenado en su buffer antes de detenerse.

[2.6] Se ha seleccionado el servicio AWS CloudFront para dar respuesta a este apartado del ejercicio 2. AWS CloudFront ofrece los siguientes modos de distribución:

·       Streaming de Baja Latencia. Para aplicaciones que requieren un retraso mínimo, utilizando AWS Elemental MediaStore y MediaLive para lograr una transmisión en tiempo real eficiente.

·       Streaming OTT (Over-The-Top). Permite la entrega de contenido de video directamente a los usuarios a través de Internet sin necesidad de una infraestructura de red dedicada, utilizando servicios como AWS Elemental MediaPackage y MediaLive.

·       Streaming Interactivo. Facilita la transmisión en vivo con interacción en tiempo real, combinando CloudFront con AWS Elemental MediaLive y servicios de señalización como Amazon Chime para lograr una experiencia interactiva.

·       Streaming Adaptativo (Adaptive Streaming). CloudFront es compatible con varios formatos de streaming adaptativo como HLS (HTTP Live Streaming) y DASH (Dynamic Adaptive Streaming over HTTP), lo que permite una transmisión de video de alta calidad que se adapta automáticamente a la velocidad de conexión de los usuarios.       

Tomando el modo de distribución streaming interactivo, se describe el proceso:

1. Preparación del contenido y configuración del origen

·       Codificación del video en vivo con AWS Elemental MediaLive, que proporciona una interfaz para configurar canales de codificación en vivo. Desde el panel de control:

o   Crear un nuevo canal de codificación en vivo seleccionando "Create Channel".

o   Configurar los parámetros de entrada, como el tipo de entrada (RTMP, HLS, etc.) y la ubicación del origen del video.

o   Definir los ajustes de video y audio, como el códec, la resolución, la tasa de bits, etc.

o   Establecer las opciones de salida, como el formato de salida (HLS, MPEG-DASH, etc.), la resolución de salida, la tasa de bits de salida, etc.

o   Guardar la configuración y activar el canal para iniciar la codificación en vivo.

o   Una vez que el canal esté activo, obtener la URL del flujo de salida para configurar CloudFront como el origen de distribución.

 

·       Configuración de Amazon Chime para la interacción en tiempo real, que permite la creación de reuniones en línea con funciones interactivas como chat, preguntas y respuestas en tiempo real. Desde su consola:

o   Crear una nueva reunión específica para el evento interactivo, definiendo la fecha, hora y duración.

o   Configurar las opciones de la reunión, como la opción de audio y video, el límite de participantes, etc.

o   Compartir los detalles de la reunión con los participantes, ya sea a través de un enlace de invitación o mediante la integración de Amazon Chime con la aplicación o plataforma.

 

2. Configuración de Amazon CloudFront

·       Creación de una Distribución de CloudFront. Desde su consola:

o   "Create Distribution" para comenzar a crear una nueva distribución.

o   Seleccionar "Web" como tipo de distribución.

o   En el campo "Origin Settings", seleccionar "MediaLive" como el origen del contenido en vivo y proporciona la URL del flujo de salida del canal de MediaLive.

o   Configurar las opciones de distribución, como el dominio alternativo (CNAME), el método de caché, los métodos HTTP permitidos, etc.

o   Establecer las políticas de seguridad, como HTTPS obligatorio, restricciones de geolocalización, etc.

o   Revisar la configuración y pulsar "Create Distribution" para crear la distribución de CloudFront.

o   Una vez que la distribución esté creada, se podrá acceder a la URL de CloudFront para distribuir el contenido de video en vivo a los espectadores.

 

3. Políticas y Roles de Gestión

·       Las políticas de usuarios y recursos se pueden gestionar con AWS IAM, que controla el acceso a los recursos de MediaLive, CloudFront y Amazon Chime.

o   Se definen permisos específicos a los usuarios o grupos de usuarios para acceder y administrar los recursos de AWS Elemental MediaLive, Amazon CloudFront y Amazon Chime. Se pueden especificar acciones permitidas o denegadas

 

·       CloudWatch para monitorear el rendimiento de los servicios y CloudTrail para auditar y registrar las actividades de los usuarios y los cambios en la configuración.

 

4. Costes Estimados:

·       CloudFront:

o   Transferencia de datos, para 10000 usuarios simultáneos, con una tasa de bits de 5 Mbps, durante 1 minuto:

v 10000 usuarios x 5 Mbps = 50000 Mbps = 50 Gbps

v 50 Gbps x 60 segundos = 3000 GB de datos transferidos

v El precio aproximado en Europa es de 0.085  por GB

v Coste de transferencia de datos: 3000 GB x 0.085  = 255 

o   Solicitudes HTTP/HTTPS:

v Aproximadamente 0.0096  por cada 10000 solicitudes.

v En este caso, asumiendo una solicitud continua para la duración del stream (1 minuto), con 10000 usuarios: 0.0096  (0.01 )

·       AWS Elemental MediaLive:

o   Costo por hora de canal activo: 0.5 

o   Para 1 minuto de transmisión: 0.5  / 60 minutos = 0.0083  por minuto (0.01 )

·       2. Amazon Chime:

o   Si se elige un plan de pago que tiene un coste de 0.1  por usuario concurrente por minuto.

o   Para 10000 usuarios concurrentes durante 1 minuto: 10000 usuarios x 0.1  = 1000 .

 

Total Estimado: 255  + 0.01  + 0.01  + 1000 = 1255.02 
